문서 객체 모델 (DOM, Documet Object Model)

웹 브라우저에 표시되는 HTML 문서 구조를 객체화 한 모델 구조.
-> 웹 브라우저는 생성한 문서 객체모델을 통해 HTML 문서의 구성요소를 객체로 인식하고,
-> 자바스크립트로 하여금 웹 브라우저의 문서 객체 모델을 조작해 웹 브라우저에 표시되는 HTML 문서 구조를 변경하거나, 새로운 구성 요소를 추가하는 등의 작업을 할 수 있다.


문서 객체 모델이 생성되는 방식

1. 웹 브라우저가 HTML 문서를 해석한다.
2. 해석한 HTML 문서 구조를 객체로 변환한다.

따라서 웹 브라우저에 표시되는 HTML 문서는 내부적으로 문서 객체 모델을 해석해서 보이게 된다.

이때, 문서 객체 모델은 나무를 뒤집어 놓은 형태의 자료구조인 트리 구조를 가지게 된다. 이를 DOM 트리라고 한다.

DOM 트리 예시

DOM 트리는 document 객체 하위에 HTML 태그 요소, 속성, 텍스트, 주석 등이 트리 형태로 구성되는데 이들을 각각 노드(node)라고 한다. 그리고 DOM 트리의 가장 꼭대기(최상위)에 있는 노드를 루트 노드(root node)라고 한다.
위 예시에서 document는 노드가 아닌 객체이므로, 루트 노드는 html이다.

각 노드는 부모, 자식, 형제관계가 형성된다.
html 노드는 head 노드와 body 노드의 부모 노드 이며, head 노드와 body 노드는 html의 자식노드이면서, 같은 레벨에 있으므로 형제 노드이다.


노드 타입 살펴보기

HTML 문법에서도 태그, 속성, 주석이 있는 것처럼 노드도 HTML 구성 요소에 따라 서로 다른 타입이 된다.

주요 노드 타입
타입 설명
문서 노드 (Node.DOCUMENT_NODE) 최상위 document 객체의 노드 타입
요소 노드 (Node.ELEMENT_NODE) h1, p 태그와 같은 요소의 노드 타입
속성 노드(Node.ATTRIBUTE_NODE) herf, src와 같은 속성의 노드 타입
텍스트 노드(Node.TEXT_NODE) 텍스트에 해당하는 노드 타입
주석 노드(Node.COMMENT_NODE) 주석에 해당하는 노드 타입

문서 객체 모델은 CSS를 포함하지 않기 때문에, CSS와 관련된 노드 타입은 없다.
